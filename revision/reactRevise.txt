1.what is React?

React is a popular JavaScript library used to build user interfaces, especially for single-page applications.
It was developed by Meta and helps developers create fast and interactive web applications.
React works with reusable components, which makes code easier to manage and maintain.
It uses a virtual DOM to update only the necessary parts of the webpage, which improves performance.
Because of its simplicity and efficiency, React is widely used in modern frontend development.


2.Library vs Framework

A library is a collection of pre-written functions or tools that developers can use whenever needed.
In a library, the developer controls the flow of the application and calls the library functions manually.
A framework, on the other hand, provides a complete structure for application development.
In a framework, the framework controls the flow and calls our code when required.
So, the main difference is that in a library we control the code, while in a framework the framework controls the code.

3.Why did you choose React?

I chose React because it is one of the most popular and in-demand frontend technologies in the industry.
It makes UI development simple by using reusable components and a structured approach.
I found React easy to learn compared to many other frontend frameworks.
Its Virtual DOM and fast rendering make applications efficient and user-friendly.
Also, React has a huge community and many job opportunities, which made it a good choice for my career.

4. what is SPA?

SPA stands for Single Page Application.
It is a type of web application where only one HTML page is loaded initially, and after that content updates dynamically 
without reloading the full page.
This makes the application faster and provides a smooth user experience.
React is commonly used to build SPAs because it can update UI components efficiently.
Examples of SPA are Gmail, Facebook, and many dashboard applications.


5. what is Virtual DOM ?

Virtual DOM is a lightweight copy of the real DOM maintained by React in memory.
Whenever data changes, React first updates the Virtual DOM instead of directly changing the real DOM.
Then React compares the new Virtual DOM with the previous one using a diffing process.
After finding the changed part, it updates only that specific part in the real DOM.
This selective update makes React applications faster and more efficient.

6.Real DOM vs Virtual DOM ?

Real DOM is the actual DOM structure present in the browser, and any change in it directly affects the webpage.
Updating the Real DOM is slower because the browser may need to re-render a large part of the page.
Virtual DOM is a lightweight copy of the Real DOM created by React in memory.
React first updates the Virtual DOM, compares it with the previous version, and then changes only the
required part in the Real DOM.
That is why Virtual DOM provides better speed and performance.


7.what is Reconciliation ?

Reconciliation is the process used by React to update the UI efficiently whenever data changes.
When a component’s state or props changes, React creates a new Virtual DOM.
Then it compares the new Virtual DOM with the previous Virtual DOM using a diffing algorithm.
After finding the differences, React updates only the changed elements in the Real DOM.
This complete comparison and update process is called Reconciliation.



8. What is Declarative UI?

Declarative UI means we describe what the user interface should look like for a given state, 
and React handles how to update it.
In React, we do not manually manipulate the DOM step by step.
We simply define the UI based on data or state.
Whenever the state changes, React automatically updates the UI accordingly.
This makes code cleaner, easier to read, and easier to maintain.


9. what is Component Based Architecture?

Component-Based Architecture means building the user interface by dividing it into small, 
independent, and reusable pieces called components.
Each component handles its own logic, structure, and sometimes styling.
These components can be combined together to create a complete application.
This approach makes the code modular, reusable, and easy to maintain.
React follows this architecture, which helps developers build large applications efficiently.



10.Why React better than vanilla JS?

React is better than Vanilla JavaScript for large and dynamic applications because it provides a 
structured and component-based approach.
In Vanilla JS, manual DOM manipulation becomes complex and hard to manage as the project grows.
React uses Virtual DOM, which makes UI updates faster and more efficient.
It also offers reusable components, state management, and cleaner code organization.
Because of this, development becomes faster, scalable, and easier to maintain in React.


11.What is JSX ?

JSX stands for JavaScript XML.
It is a syntax extension in React that allows us to write HTML-like code inside JavaScript.
JSX makes the UI code more readable, simple, and easy to understand.
Browsers do not understand JSX directly, so React uses Babel to convert JSX into normal JavaScript.
With JSX, creating dynamic and interactive UI becomes much easier.


12.JSX ka syntax ?

function App() {
  let name = "Hario";

  return (
    <h1>Welcome {name}</h1>
  );
}


13. JSX rules?

JSX has some specific rules that must be followed while writing code in React.
First, JSX must return only one parent element.
Second, all tags should be properly closed like XML.
Third, JavaScript expressions are written inside curly braces {}.
Also, some HTML attributes have different names in JSX such as className instead of class 
and htmlFor instead of for.

14.Single parent element?

In JSX, a component must return only one parent element.
This means all JSX elements should be wrapped inside a single root container like a div or <> </> fragment.
React expects one root node because JSX gets converted into a single JavaScript object.
If we return multiple sibling elements without wrapping, React throws an error.
That is why a single parent element is required.


15.why use className?

In React JSX, we use className instead of class to apply CSS classes.
This is because class is a reserved keyword in JavaScript used for creating classes.
Since JSX is written inside JavaScript, using class directly can create confusion.
So React introduced className as an alternative attribute.
Its function is the same as the HTML class attribute.


16.htmlFor kyu use hota hai?

In React JSX, we use htmlFor instead of for inside the label tag.
This is because for is a reserved keyword in JavaScript, mainly used in loops.
Since JSX is written inside JavaScript, React cannot use for directly as an attribute name.
So React provides htmlFor as the alternative.
Its purpose is to connect a label with an input element.

17.JS expression inside JSX?

In JSX, we can write JavaScript expressions inside curly braces {}.
This allows us to display dynamic values, variables, calculations,
conditions, or function results inside the UI.
React evaluates the expression and renders its output on the screen.
It makes JSX powerful because HTML and JavaScript can work together.
This feature is widely used for dynamic content rendering.


18.Conditional rendering in JSX ?

Conditional rendering means showing different UI elements based on a condition.
In React JSX, we use JavaScript expressions like ternary operator, logical AND, 
or if conditions to render content dynamically.
It helps us display specific components or messages depending on data or state.
For example, showing “Welcome” when user is logged in and “Login First” when user is not logged in.
This makes the UI interactive and responsive.

<h1>{10 > 5 ? "Yes" : "No"}</h1>


19.Ternary operator?

The ternary operator is a short form of if-else used to check conditions in a single line.
It contains three parts: condition, true value, and false value.
If the condition is true, the first value is returned; otherwise, the second value is returned.
In React JSX, ternary operator is commonly used for conditional rendering.
It makes the code shorter and cleaner.

20.&& operator?

The && operator is called the logical AND operator.
In React JSX, it is used when we want to render something only if the condition is true.
If the condition is true, the JSX after && gets displayed.
If the condition is false, React renders nothing.
It is useful for showing optional content like messages, buttons, or alerts.

function App() {
  let isAdmin = true;

  return (
    <div>
      {isAdmin && <h1>Admin Panel</h1>}
    </div>
  );
}


21.map() in JSX ?

In React JSX, map() is used to render multiple elements dynamically from an array.
It loops through each item of the array and returns JSX for every item.
This helps us display lists like users, products, cards, or menu items easily.
Instead of writing repeated HTML manually, map() creates UI automatically.
So map() is mainly used for list rendering in React.

function App() {
  let arr = ["Ram", "Shyam", "Mohan"];

  return (
    <div>
      {arr.map((item) => <h1>{item}</h1>)}
    </div>
  );
}


22. what is  Components?

Components are small, reusable, independent pieces of UI in React.
Each component is responsible for rendering a specific part of the application,
such as a button, navbar, card, or form.
Instead of writing the whole UI in one file, React divides it into components.
These components can be reused multiple times wherever needed.
This makes the code modular, clean, and easy to maintain.


23. Functional Component?

A Functional Component is a simple JavaScript function that returns JSX.
It is used to create UI elements in React.
Functional components are easy to write, easy to understand, and widely used in modern React applications.
They can also use React Hooks for state and lifecycle features.
Because of simplicity and better readability, functional components are preferred over class components.

function Welcome() {
  return <h1>Hello Hario</h1>;
}


24. Class Component (basic only, interview ke liye knowledge)?

A Class Component is a React component created using an ES6 class.
It extends React.Component and must contain a render() method that returns JSX.
Before Hooks were introduced, class components were mainly used to manage state and lifecycle methods.
They are more complex compared to functional components.
Nowadays, class components are less preferred, but basic knowledge is still important.

import React, { Component } from "react";

class Welcome extends Component {
  render() {
    return <h1>Hello User</h1>;
  }
}


25.Reusable components ?

Reusable components are components that can be used multiple times in different parts of the application without rewriting the same code.
We create a component once and call it wherever needed.
This reduces code duplication and saves development time.
Reusable components also make the application easier to maintain and update.
React strongly supports this concept through component-based architecture.


26 Nested components?

Nested components mean placing one component inside another component.
In React, a parent component can contain one or more child components inside its JSX.
This helps us build complex UI structures in a modular way.
Each small child component handles its own part, and together they form the complete page.
This makes the code more organized and manageable. 


27.Export/Import Components?

In React, export and import are used to share components between different files.
We create a component in one file and export it, then import it wherever we want to use it.
This helps in organizing code into separate reusable files.
React mainly uses two types: default export and named export.
Using export/import makes the project clean and modular.


28.Component Naming Convention ?

In React, component names should always start with a capital letter.
This is the standard naming convention used to differentiate components from normal HTML tags.
If a component name starts with a small letter, React treats it like a regular HTML element.
Component names should also be meaningful and related to their purpose, like Navbar, LoginForm, or ProductCard.
Following naming conventions improves readability and project structure.



29. what is Props?

Props stands for Properties.
Props are used to pass data from a parent component to a child component in React.
They make components dynamic and reusable because the same component can display different data.
Props are read-only, which means the child component cannot directly modify them.
In simple words, props are like arguments passed to components.

30.Parent to child data pass?

In React, parent to child data passing is done using props.
The parent component sends data as attributes while calling the child component.
The child component receives that data inside the props object.
This is the standard way of communication from parent to child in React.
It follows one-way data flow.


31.Multiple props?

In React, we can pass multiple props from parent to child just like multiple attributes in HTML.
Each prop carries different data such as name, age, city, role, etc.
The child component receives all these values inside the props object.
This makes the component more dynamic and flexible.
It is commonly used when one component needs multiple pieces of information.


32.Props destructuring ?

Props destructuring means extracting prop values directly from the props object.
Instead of writing props.name, props.age, props.city again and again, we can destructure them into variables.
This makes the code shorter, cleaner, and more readable.
It is a common modern React practice.
Destructuring improves code maintainability


33.what is children props ?

children props is a special built-in prop in React.
It is used to access whatever content is written between the opening and closing tags of a component.
This allows us to create wrapper or reusable layout components.
Instead of passing data as normal props, we can directly nest JSX inside the component.
React automatically stores that nested content inside props.children.

34.what is Default props?

Default props are predefined values assigned to props when no value is passed from the parent component.
They work as fallback values.
If the parent sends a prop, React uses that value; otherwise, it uses the default one.
This helps avoid undefined values and makes components safer.
Default props make components more reliable and reusable.


35.Props drilling kya hota hai?

Props drilling is the process of passing data from a parent component to a deeply nested child 
component through multiple intermediate components.
Even if the middle components do not need that data, they still have to pass it forward.
This makes the code lengthy, difficult to manage, and less maintainable.
Props drilling usually happens in large React applications.
To solve this, developers use Context API or state management libraries.



36. what is Event Handling?

Event Handling in React means responding to user actions such as button click, input typing, form submit, mouse hover, etc.
React provides built-in event handlers to manage these actions.
We handle an event by attaching a function to an element.
When the event occurs, that function gets executed.
Event handling makes the application interactive and dynamic.


✅ onClick

onClick event is used when we want to perform an action after clicking on an element like a button, link, or icon.
When the user clicks, the attached function gets executed.

✅ onChange

onChange event is used to detect changes in input fields, textarea, or select box.
Whenever the user types or changes the value, the function runs.

✅ onSubmit

onSubmit event is used with forms.
When the user submits the form, the attached function gets executed.

✅ onMouseEnter

onMouseEnter event is triggered when the mouse pointer enters an element.
It is mostly used for hover effects.

37.Function call in event ?

Function call in event means executing a function when a particular event occurs in React.
We attach a function with event handlers like onClick, onChange, or onSubmit.
When the user performs that action, React automatically calls the function.
This function can contain any logic like alert, data update, or state change.
It is used to connect user interaction with application behavior.


38.Passing parameter in event ?

Passing parameter in event means sending a value to a function when an event is triggered.
Sometimes we need to pass custom data like id, name, or message during a button click or other event.
In React, we cannot call the function directly with parameters inside the event handler.
So we use an arrow function to pass the parameter.
This allows us to execute the function with specific values when the event happens.


39.preventDefault()?

preventDefault() is a JavaScript method used to stop the default behavior of an event.
In React, it is mostly used in form submission to prevent the page from reloading.
Normally when a form is submitted, the browser refreshes automatically.
By using preventDefault(), we can handle the form data manually with JavaScript.
This gives us full control over the form submission process.


40. State (Very Important) ?

State is a built-in object in React used to store dynamic data inside a component.
When the state value changes, React automatically re-renders the component UI.
State is used for data that changes over time, such as input values, counters, 
user actions, or API data.
Unlike props, state is managed inside the component itself.
State makes the UI interactive and dynamic.


41.Why useState?

useState is a React Hook used to create and manage state inside functional components.
Before hooks, functional components could not store changing data.
With useState, we can store dynamic values like counters, input text, toggles, etc.
Whenever the state updates, React automatically updates the UI.
That is why useState is used to make functional components interactive.

✅ Syntax
const [state, setState] = useState(initialValue);

✅ Meaning
state = current value
setState = update function
initialValue = starting value


✅ Updating State

Updating state means changing the current value stored in state.
In React, we never change the state variable directly.
We always use the setter function provided by useState.
When the setter function runs, React updates the value and re-renders the component.
This keeps the UI in sync with the latest data.
Async nature of state update

✅ Previous State Callback

Previous state callback is used when the new state depends on the old state value.
Instead of directly writing setCount(count + 1), we use a callback function.
React passes the previous state value into that callback.
This ensures that the state updates correctly even if multiple updates happen quickly.
It is the safer way to update dependent state values.

✅ Example
setCount((prev) => prev + 1);


✅ Multiple States

✅ Example
const [name, setName] = useState("Hario");
const [age, setAge] = useState(22);

Multiple states mean creating more than one state variable inside the same component.
We use separate useState hooks for different pieces of data.
For example, one state for name, one for age, and one for count.
This keeps each data independent and easier to manage.
React allows unlimited state variables in one component.

✅ Object State Update

Sometimes we store multiple related values inside one object state.
While updating object state, we should not overwrite the whole object accidentally.
So we use the spread operator to keep old values and update only the required property.
This helps preserve the remaining data.
It is the correct way to update object state in React.

✅ Example
const [user, setUser] = useState({name:"Hario", age:22});

setUser({...user, age:23});


✅ Array State Update

When state stores an array, we should update it by creating a new array.
React state should not be modified directly using push or pop.
We use spread operator, map, filter, or other array methods to create updated arrays.
Then we pass that new array to the setter function.
This helps React detect changes properly.

✅ Example
const [arr, setArr] = useState([1,2,3]);

setArr([...arr, 4]);
